# EP-00001 â€” yt-mp3: YouTube audio downloader (first tool)

## Problem / Pain Points

- The `simple-tools` repo has a SPECS.md but no actual code: no `pyproject.toml`, no `src/`, no entrypoint, no `.venv`. We need both the project skeleton and the first tool, in that order.
- The first tool, `yt-mp3`, takes a YouTube URL and produces an MP3 of its audio track on disk. End user goal: paste a URL, get a usable MP3 in `output/yt-mp3/`.
- Constraints from SPECS.md that shape this work:
  - One Typer entrypoint (`simple-tools`) with subcommands.
  - Each tool is a package directory under `src/simple_tools/tools/`.
  - Stdout is reserved for the final artifact path; progress goes to stderr.
  - No network calls in tests.
  - System dep `ffmpeg` is a runtime requirement (not vendored).

## Suggested Solution

### Project skeleton (one-time)

- Root `pyproject.toml` defining package `simple-tools`, Python `>=3.12`, deps (`typer`, `yt-dlp`), dev deps (`pytest`, `mypy`), and `[project.scripts] simple-tools = "simple_tools.cli:app"`.
- `.python-version` pinning a 3.12.x interpreter.
- `.gitignore` covering `.venv/`, `output/`, `__pycache__/`, `.pytest_cache/`, `.mypy_cache/`, `*.egg-info/`, `.idea/`.
- `src/simple_tools/__init__.py` and `src/simple_tools/cli.py` with a top-level Typer `app` and a help-only entry until subcommands are registered.
- `src/simple_tools/tools/__init__.py` (empty namespace).

### yt-mp3 tool layout

```
src/simple_tools/tools/yt_mp3/
â”śâ”€â”€ __init__.py        # re-exports the Typer command/app
â”śâ”€â”€ cli.py             # flag parsing, exit codes, stdout/stderr discipline
â””â”€â”€ downloader.py      # yt-dlp wrapper; pure function, easy to mock
```

- `cli.py` registers a single command `yt-mp3` on the root app (via `@app.command("yt-mp3")` in the root, or `app.add_typer` with a one-command sub-app â€” pick whichever is cleaner once code exists).
- `downloader.py` exposes one function, e.g. `download_mp3(url: str, output_dir: Path, bitrate: int, filename: str | None) -> Path`, returning the final file path. Uses `yt_dlp.YoutubeDL` with `format="bestaudio/best"` and a `FFmpegExtractAudio` postprocessor configured to MP3 at the requested bitrate.
- Filename: when `--filename` is omitted, use the video's sanitized title (yt-dlp's `outtmpl` with `restrictfilenames=True`). When provided, use it verbatim (still strip path separators).
- Output dir: created with `mkdir(parents=True, exist_ok=True)`. Default `output/yt-mp3` resolved relative to CWD.
- Overwrite policy: if the target file already exists, exit non-zero with a clear stderr message. No `--force` in v1.
- Error handling: catch `yt_dlp.utils.DownloadError` and any `OSError` at the CLI boundary, print one-line error to stderr, exit `1`. With `--debug`, re-raise so Typer/Python show the traceback.
- Progress: yt-dlp prints to stderr by default; keep that. Final line on stdout is the absolute path of the produced MP3.
- URL validation v1: trust yt-dlp to reject bad URLs (fail fast on the download attempt). No separate pre-flight HEAD request.
